I'm also making this a community wiki in case others want to add to or correct these observations.</p>\u000a\u000a<p>My original reduce statement is redundant and is better written this way:</p>\u000a\u000a<pre><code>&gt;&gt;&gt; reduce(list.__add__, (list(mi) for mi in list_of_menuitems))\u000a</code></pre>\u000a\u000a<p>This is the correct syntax for a nested list comprehension (Brilliant summary <a href="http://stackoverflow.com/users/3002/df">dF</a>!):</p>\u000a\u000a<pre><code>&gt;&gt;&gt; [image for mi in list_of_menuitems for image in mi]\u000a</code></pre>\u000a\u000a<p>But neither of these methods are as efficient as using <code>itertools.chain</code>:</p>\u000a\u000a<pre><code>&gt;&gt;&gt; from itertools import chain\u000a&gt;&gt;&gt; list(chain(*list_of_menuitems))\u000a</code></pre>\u000a\u000a<p>And as @cdleary notes, it's probably better style to avoid * operator magic by using <code>chain.from_iterable</code> like so:</p>\u000a\u000a<pre><code>&gt;&gt;&gt; chain = itertools.chain.from_iterable([[1,2],[3],[5,89],[],[6]])\u000a&gt;&gt;&gt; print(list(chain))\u000a&gt;&gt;&gt; [1, 2, 3, 5, 89, 6]\u000a</code></pre>\u000a
