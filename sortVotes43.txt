V<p>As I only now noticed after commenting on <a href="http://stackoverflow.com/questions/6900955/python-convert-list-to-dictionary/6900977#6900977">this answer</a>, slices in Python 3 return shallow copies of whatever they're slicing rather than views. Why is this still the case? Even leaving aside numpy's usage of views rather than copies for slicing, the fact that <code>dict.keys</code>, <code>dict.values</code>, and <code>dict.items</code> all return views in Python 3, and that there are many other aspects of Python 3 geared towards greater use of iterators, makes it seem that there would have been a movement towards slices becoming similar. <code>itertools</code> does have an <code>islice</code> function that makes iterative slices, but that's more limited than normal slicing and does not provide view functionality along the lines of <code>dict.keys</code> or <code>dict.values</code>.</p>\u000a\u000a<p>As well, the fact that you can use assignment to slices to modify the original list, but slices are themselves copies and not views, is a contradictory aspect of the language and seems like it violates several of the principles illustrated in <a href="http://www.python.org/dev/peps/pep-0020/" rel="noreferrer">the Zen of Python</a>.</p>\u000a\u000a<p>That is, the fact you can do</p>\u000a\u000a<pre><code>&gt;&gt;&gt; a = [1, 2, 3, 4, 5]\u000a&gt;&gt;&gt; a[::2] = [0, 0, 0]\u000a&gt;&gt;&gt; a\u000a[0, 2, 0, 4, 0]\u000a</code></pre>\u000a\u000a<p>But not</p>\u000a\u000a<pre><code>&gt;&gt;&gt; a = [1, 2, 3, 4, 5]\u000a&gt;&gt;&gt;
