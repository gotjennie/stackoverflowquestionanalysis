It's a result both comforting and unexpected.</p>\u000a\u000a<p><strong>Is there another, "better" way of traversing a list in pairs?</strong></p>\u000a\u000a<p>Note that if the list has an odd number of elements then the last one will not be in any of the pairs. </p>\u000a\u000a<p><strong>Which would be the right way to ensure that all elements are included?</strong></p>\u000a\u000a<p>I added these two suggestions from the answers to the tests:</p>\u000a\u000a<pre><code>def pairwise(t):\u000a    it = iter(t)\u000a    return izip(it, it)\u000a\u000adef chunkwise(t, size=2):\u000a    it = iter(t)\u000a    return izip(*[it]*size)\u000a</code></pre>\u000a\u000a<p>These are the results:  </p>\u000a\u000a<pre><code>0.00159502029419 1.25745987892 1.25586485863\u000a0.00222492218018 1.23795199394 1.23572707176\u000a</code></pre>\u000a\u000a<h2>Results so far</h2>\u000a\u000a<p>Most pythonic and very efficient:</p>\u000a\u000a<pre><code>pairs = izip(t[::2], t[1::2])\u000a</code></pre>\u000a\u000a<p>Most efficient and very pythonic:</p>\u000a\u000a<pre><code>pairs = izip(*[iter(t)]*2)\u000a</code></pre>\u000a\u000a<p>It took me a moment to grok that the first answer uses two iterators while the second uses a single one.</p>\u000a\u000a<p>To deal with sequences with an odd number of elements, the suggestion has been to augment the original sequence adding one element (<code>None</code>) that gets paired with the previous last element, something that can be achieved with <code>itertools.izip_longest()</code>.</p>\u000a\u000a<h2>Finally</h2>\u000a\u000a<p>Note that, in Python 3.x, <code>zip()</code> behaves as <code>itertools.izip()</code>, and <code>itertools.izip()</code>  is gone.</p>\u000a
