V<p>I'm using PIL to convert a transparent PNG image uploaded with Django to a JPG file. The output looks broken.</p>\u000a\u000a<h3>Source file</h3>\u000a\u000a<p><img src="https://i.stack.imgur.com/I2uNe.png" alt="transparent source file"></p>\u000a\u000a<h3>Code</h3>\u000a\u000a<pre><code>Image.open(object.logo.path).save('/tmp/output.jpg', 'JPEG')\u000a</code></pre>\u000a\u000a<p>or</p>\u000a\u000a<pre><code>Image.open(object.logo.path).convert('RGB').save('/tmp/output.png')\u000a</code></pre>\u000a\u000a<h3>Result</h3>\u000a\u000a<p>Both ways, the resulting image looks like this:</p>\u000a\u000a<p><img src="https://i.stack.imgur.com/rmlPz.png" alt="resulting file"></p>\u000a\u000a<p>Is there a way to fix this? I'd like to have white background where the transparent background used to be.</p>\u000a\u000a<hr>\u000a\u000a<h1>Solution</h1>\u000a\u000a<p>Thanks to the great answers, I've come up with the following function collection:</p>\u000a\u000a<pre><code>import Image\u000aimport numpy as np\u000a\u000a\u000adef alpha_to_color(image, color=(255, 255, 255)):\u000a    """Set all fully transparent pixels of an RGBA image to the specified color.\u000a    This is a very simple solution that might leave over some ugly edges, due\u000a    to semi-transparent areas. You should use alpha_composite_with color instead.\u000a\u000a    Source: http://stackoverflow.com/a/9166671/284318\u000a\u000a    Keyword Arguments:\u000a    image -- PIL RGBA Image object\u000a    color -- Tuple r, g, b (default 255, 255, 255)\u000a\u000a    """ \u000a    x = np.array(image)\u000a    r, g, b, a = np.rollaxis(x, axis=-1)\u000a    r[a == 0] = color[0]\u000a    g[a == 0] = color[1]\u000a    b[a == 0] = color[2] \u000a    x = np.dstack([r, g, b, a])\u000a    return Image.fromarray(x, 'RGBA')\u000a\u000a\u000adef alpha_composite(front, back):\u000a    """Alpha composite two RGBA images.\u000a\u000a    Source: http://stackoverflow.com/a/9166671/284318\u000a\u000a    Keyword Arguments:\u000a    front -- PIL RGBA Image object\u000a    back -- PIL RGBA Image object\u000a\u000a    """\u000a    front = np.asarray(front)\u000a    back = np.asarray(back)\u000a    result = np.empty(front.shape, dtype='float')\u000a    alpha = np.index_exp[:, :, 3:]\u000a    rgb = np.index_exp[:, :, :3]\u000a    falpha = front[alpha] / 255.0\u000a    balpha = back[alpha] / 255.0\u000a    result[alpha] = falpha + balpha * (1 - falpha)\u000a    old_setting = np.seterr(invalid='ignore')\u000a    result[rgb] = (front[rgb] * falpha + back[rgb] * balpha * (1 - falpha)) / result[alpha]\u000a    np.seterr(**old_setting)\u000a    result[alpha] *= 255\u000a    np.clip(result, 0, 255)\u000a    # astype('uint8') maps np.nan and np.inf to 0\u000a    result = result.astype('uint8')\u000a    result = Image.fromarray(result, 'RGBA')\u000a    return result\u000a\u000a\u000adef alpha_composite_with_color(image, color=(255, 255, 255)):\u000a    """Alpha composite an RGBA image with a single color image of the\u000a    specified color and the same size as the original image.\u000a\u000a    Keyword Arguments:\u000a    image -- PIL RGBA Image object\u000a    color -- Tuple r, g, b (default 255, 255, 255)\u000a\u000a    """\u000a    back = Image.new('RGBA', size=image.size, color=color + (255,))\u000a    return alpha_composite(image, back)\u000a\u000a\u000adef pure_pil_alpha_to_color_v1(image, color=(255, 255, 255)):\u000a    """Alpha composite an RGBA Image with a specified color.\u000a\u000a    NOTE: This version is much slower than the\u000a    alpha_composite_with_color solution. Use it only if\u000a    numpy is not available.\u000a\u000a    Source: http://stackoverflow.com/a/9168169/284318\u000a\u000a    Keyword Arguments:\u000a    image -- PIL RGBA Image object\u000a    color -- Tuple r, g, b (default 255, 255, 255)\u000a\u000a    """ \u000a    def blend_value(back, front, a):\u000a        return (front * a + back * (255 - a)) / 255\u000a\u000a    def blend_rgba(back, front):\u000a        result = [blend_value(back[i], front[i], front[3]) for i in (0, 1, 2)]\u000a        return tuple(result + [255])\u000a\u000a    im = image.copy()  # don't edit the reference directly\u000a    p = im.load()  # load pixel array\u000a    for y in range(im.size[1]):\u000a        for x in range(im.size[0]):\u000a            p[x, y] = blend_rgba(color + (255,), p[x, y])\u000a\u000a    return im\u000a\u000adef pure_pil_alpha_to_color_v2(image, color=(255, 255, 255)):\u000a    """Alpha composite an RGBA Image with a specified color.\u000a\u000a    Simpler, faster version than the solutions above.\u000a\u000a    Source: http://stackoverflow.com/a/9459208/284318\u000a\u000a    Keyword Arguments:\u000a    image -- PIL RGBA Image object\u000a    color -- Tuple r, g, b (default 255, 255, 255)\u000a\u000a    """\u000a    image.load()  # needed for split()\u000a    background = Image.new('RGB', image.size, color)\u000a    background.paste(image, mask=image.split()[3])  # 3 is the alpha channel\u000a    return background\u000a</code></pre>\u000a\u000a<h2>Performance</h2>\u000a\u000a<p>The simple non-compositing <code>alpha_to_color</code> function is the fastest solution, but leaves behind ugly borders because it does not handle semi transparent areas.</p>\u000a\u000a<p>Both the pure PIL and the numpy compositing solutions give great results, but <code>alpha_composite_with_color</code> is much faster (8.93 msec) than <code>pure_pil_alpha_to_color</code> (79.6 msec). <del>If numpy is available on your system, that's the way to go.</del> (Update: The new pure PIL version is the fastest of all mentioned solutions.)</p>\u000a\u000a<pre><code>$ python -m timeit "import Image; from apps.front import utils; i = Image.open(u'logo.png'); i2 = utils.alpha_to_color(i)"\u000a10 loops, best of 3: 4.67 msec per loop\u000a$ python -m timeit "import Image; from apps.front import utils; i = Image.open(u'logo.png'); i2 = utils.alpha_composite_with_color(i)"\u000a10 loops, best of 3: 8.93 msec per loop\u000a$ python -m timeit "import Image; from apps.front import utils; i = Image.open(u'logo.png'); i2 = utils.pure_pil_alpha_to_color(i)"\u000a10 loops, best of 3: 79.6 msec per loop\u000a$ python -m timeit "import Image; from apps.front import utils; i = Image.open(u'logo.png'); i2 = utils.pure_pil_alpha_to_color_v2(i)"\u000a10 loops, best of 3: 1.1 msec per loop\u000a</code></pre>\u000a

You simply do theList.sort().\u000aThis is O(n * log(n)) string comparisons and no memory management (since all\u000astrings and list elements are some sort of smart pointers). You are happy.</p>\u000a\u000a<p>Now, you want to do the same, but ignore the case (let's simplify and say\u000aall strings are ascii, so locale issues can be ignored).\u000aYou can do theList.sort(key=lambda s: s.lower()), but then you cause two new \u000aallocations per comparison, plus burden the garbage-collector with the duplicated\u000a(lowered) strings. \u000aEach such memory-management noise is orders-of-magnitude slower than simple string comparison.</p>\u000a\u000a<p>Now, with an in-place stricmp()-like function, you do: theList.sort(cmp=stricmp)\u000aand it is as fast and as memory-friendly as theList.sort(). You are happy again.</p>\u000a\u000a<p>The problem is any Python-based case-insensitive comparison involves implicit string \u000aduplications, so I was expecting to find a C-based comparisons (maybe in module string).</p>\u000a\u000a<p>Could not find anything like that, hence the question here.\u000a(Hope this clarifies the question).</p>\u000a
